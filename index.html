# make - 어떤 명령어 앞에 들여쓰기를 할 때, 꼭 space가 아닌 tab만을 사용해야 한다. - make 파일은 하나의 디렉토리에 하나만 존재해야 한다. - 보통 오브젝트는 사용하고 난 후 용도가 없어지면 clean을 이용하여 지운다. # 기타 - 컴퓨터 문서에서 []는 생략 가능 - path는 경로 - 보통 헤더파일(#include 뒤에 오는 것)은 include 안에 있음 - 리눅스에서 현재 디렉토리의 명령어를 실행할 때 앞에 './'을 붙이는 이유는 보안 때문 - 프로그램에서 디바이스를 번호로 관리하고, 이 때 그 번호는 파일 디스크립트 - 파일 디스크립트는 모든 열려있는 파일을 참조할 때 사용하고 보통 0. 키보드 (입력장치) 1. 모니터 (출력장치) 2. 에러처리 파일 3... 실행한 파일 (안 열리면 -1) - size_t: unsigned - ssize_t: signed

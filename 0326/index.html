# vi 명령어 1. 명령모드 - 현재 커서 위치에서 편집모드로 전환: i - 현재 커서 위 맨 앞 문자에서 편집모드로 전환: I - 현재 커서 바로 뒤에서 편집모드로 전환: a - 현재 커서 줄의 맨 뒤 문자에서 편집모드로 전환: A - 다음 단어 위치로 커서 이동: w - 이전 단어 위치로 커서 이동: b - 커서 줄의 첫 문자로 이동: ^(캐럿), 0 - 커서 줄의 끝 문자로 이동: $ - 마지막 라인으로 이동: G - 100번째 라인으로 이동: 100G (순서는 1 ~ n) - 방향키 → : h - 방향키 ← : l - 방향키 ↑ : k - 방향키 ↓ : j - 편집된 내용 되돌리기: u - 커서 앞 글자 삭제: X - 커서 위치 글자 삭제: x - 커서 위치 포함 세 글자 삭제: 3x (n글자, nx) - 커서 위치 단어 삭제: dw (delete word인가?) - 커서 위치 라인 삭제: dd - 커서 위치 포함 세 라인 삭제: 3dd (n개의 라인 ndd을 삭제할 때, 총 라인의 수 > n이어야 함) - 커서 위치에서 라인 마지막 문자까지 삭제: D - 커서 위치 라인을 버퍼에 복사: yy - 커서 위치 포함 세 라인을 버퍼에 복사: 3yy - 버퍼에 복사된 문자열을 커서 위치에 붙여넣기: p - 버퍼에 복사된 문자열을 커서 앞에 붙여넣기: P - 한 글자 수정: r - 커서 다음 위치에 빈 줄 추가: o - 커서 이전 위치에 빈 줄 추가: O - 커서 위치 라인과 다음 라인 합체: J 2. EX 모드 (커맨드 입력시 ':' 사용) - 변경 내용 저장: w - 변경 내용 저장 후 종료: wp - 저장 안 하고 종료: q! - 강제 종료: Q - 문자열 검색: / (예: Hello를 검색할 땐 /Hello) (이전 / 다음 검색 문자열: N / n) - 문자열 변경: %s/ / /ig (예: Hello를 Bye로 변경할 땐 $s/Hello/Bye/ig) - 라인 번호 표시: set nu - 라인 번호 비표시: set nonu 3. 비주얼 모드 - 명령모드에서 전환시 - 글자 단위로 선택: v - 라인 단위로 선택: V - 블럭 단위로 선택: ^v - 선택한 텍스트를 버퍼에 복사: y - 선택한 텍스트를 삭제: d, x # gcc 컴파일러 - 실행 전 설치 & 업데이트 방법 1) sudo apt-get update 2) sudo apt-get upgrade 3) sudo apt install gcc # 명령어 - !v: 가장 최근에 입력한 v로 시작하는 명령어 (!n: 가장 최근에 입력한 n으로 시작하는 명령어) - gcc -o 명령어 파일 (예: Hello.c 파일을 Hello 명령으로 실행하게 한다면 gcc -o Hello Hello.c) - 컴파일 gcc 파일 (Index.c 파일이면 gcc Index.c) - 실행: a.out (위의 컴파일한 파일 실행)
